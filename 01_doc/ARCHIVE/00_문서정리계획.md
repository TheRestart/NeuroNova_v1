## 🔄 Git 저장소 구조 고려사항

### 복수 저장소 관리
본 프로젝트는 **4개의 독립된 Git 저장소**로 관리됩니다:

1. **메인 저장소** (`NeuroNova_v1`): 문서 + UML ← **이번 작업 대상**
2. **백엔드 저장소** (`NeuroNova_02_back_end`): Django 코드
3. **React 프론트엔드 저장소** (`NeuroNova_03_front_end_react`): React 코드
4. **Flutter 프론트엔드 저장소** (`NeuroNova_04_front_end_flutter`): Flutter 코드

**중요:** 문서 정리 작업은 메인 저장소(`NeuroNova_v1`)의 `01_doc/` 디렉토리에만 영향을 미칩니다.

### Git 커밋 전략
- 메인 저장소에서만 커밋/푸시 수행
- 백엔드/프론트엔드 저장소는 영향 없음
- 문서 정리 완료 후 단일 커밋으로 정리
